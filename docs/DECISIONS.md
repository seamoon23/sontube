# SonTube Decisions

## D001. YouTube Data API 미사용

MVP에서는 YouTube Data API를 사용하지 않는다.

이유:

- API 비용과 쿼터 부담을 제거한다.
- Google API Key 관리가 필요 없어야 한다.
- 전체 검색과 채널 수집 유혹을 차단한다.
- 보호자 직접 검토라는 제품 철학을 유지한다.

## D002. 전체 검색 미제공

아이 화면 검색은 내부 DB에 저장된 승인 영상만 대상으로 한다.

## D003. 채널/플레이리스트 자동 수집 제외

채널이나 플레이리스트 전체가 안전하다고 보장할 수 없고, 자동 수집에는 외부 API 또는 스크래핑 유혹이 생긴다. MVP에서는 개별 영상 URL만 등록한다.

## D004. 댓글과 자막 분석 제외

댓글과 자막은 품질과 안전성 변동이 크고, 조회/분석을 위해 외부 API나 다운로드 흐름이 필요할 수 있다. MVP에서는 YouTube 플레이어 자체 기능 이상으로 다루지 않는다.

## D005. 자체 영상 업로드 제외

SonTube는 YouTube에 이미 있는 보호자 검토 영상을 정리하는 도구로 시작한다. 저장 비용, 스트리밍 구현, 권리 문제를 피하기 위해 자체 영상 업로드는 제외한다.

## D006. 추천 태그는 내부 데이터 기반

추천 태그는 최근 사용 태그, 자주 사용한 태그, 제목/설명 키워드, 같은 키워드가 포함된 기존 영상의 태그를 기반으로만 만든다. 추천 결과는 자동 적용하지 않고 보호자가 선택한다.

## D007. 아이 화면의 관리 기능 비노출

아이 화면에는 관리자 버튼, 설정 버튼, YouTube 전체 검색, 채널 이동용 UI를 두지 않는다.
